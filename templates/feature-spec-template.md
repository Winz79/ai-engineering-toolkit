## 6. User Experience / Workflow

Describe the expected workflow or interaction.

If there is no user interface, describe the API, process, or operational workflow.

## 7. Data and Contracts

Capture relevant data, API, event, file, configuration, or integration contracts.

## 8. Impact Analysis Summary

Summarize likely impact across:

- domain behavior
- existing code paths
- data or migrations
- integrations
- security and privacy
- observability
- operations
- documentation

## 9. Design Notes

Describe the proposed implementation shape at a level suitable for review.

Link ADRs when the feature requires architectural decisions.

## 10. Test Strategy

List the tests or verification steps needed.

Include current behavior characterization tests when changing legacy or poorly documented behavior.

## 11. Rollout and Recovery

Describe release sequencing, flags, migration, rollback, or support needs.
